# 成员 A 任务书：前端页面与游戏化视觉

## 1. 你的职责

你负责 LifeQuest 的前端页面和游戏化视觉效果。目标是让项目看起来像一个 RPG 人生副本，而不是普通待办事项工具。

## 2. 你需要先了解

项目核心流程：

```text
登录
→ 创建人生角色
→ 添加长期目标
→ 生成副本任务
→ 完成任务
→ 获得经验
→ 查看等级、徽章和排行榜
```

当前前端代码在：

```text
client/index.html
client/styles.css
client/app.js
```

## 3. 主要任务

### 3.1 优化首页

位置：

```text
client/index.html
client/styles.css
```

需要优化：

- 项目标题：`LifeQuest 人生副本`
- Slogan：`把现实目标变成可挑战的游戏副本`
- CTA 按钮：`开启副本`
- 功能介绍：AI 任务生成、经验升级、徽章成就、排行榜
- 视觉风格：RPG、科幻、游戏面板、发光卡片

验收标准：

- 首页第一眼有创意。
- 能截图放进 PPT。
- 不影响登录和跳转。

### 3.2 优化今日副本看板

位置：

```text
client/index.html
client/styles.css
client/app.js
```

需要展示：

- 角色昵称。
- 职业标签。
- 等级。
- XP。
- 经验条。
- 金币。
- 连击天数。
- 今日任务数量。
- 已完成数量。
- NPC 鼓励语。

验收标准：

- 这是答辩最重要页面，必须完整美观。
- XP 和等级数据来自接口，不要写死。
- 页面尺寸适合截图。

### 3.3 优化任务卡片

需要区分：

```text
主线任务
支线任务
每日任务
Boss 任务
```

建议样式：

- 主线任务：紫色。
- 支线任务：蓝色。
- 每日任务：绿色。
- Boss 任务：金色或红色。

每张卡片展示：

- 任务标题。
- 任务描述。
- 任务类型。
- 任务难度。
- XP 奖励。
- 完成按钮。

验收标准：

- 点击完成后按钮状态变化。
- 已完成任务和未完成任务有明显区别。

### 3.4 优化徽章墙

需要展示：

- 徽章图标。
- 徽章名称。
- 解锁说明。
- 已解锁和未解锁状态。

验收标准：

- 已解锁徽章高亮。
- 未解锁徽章置灰。
- 页面可以截图展示成就系统。

### 3.5 优化排行榜

需要展示：

- 排名。
- 用户昵称。
- 职业。
- 等级。
- XP。

验收标准：

- 前三名有特殊视觉。
- 数据来自 `/api/ranking`。

## 4. 不建议你做

- 不要改数据库结构。
- 不要改 API 路径。
- 不要重写后端。
- 不要删除组长已有核心逻辑。

## 5. 你的交付物

- 修改后的前端页面。
- 页面截图。
- 简短说明：完成了哪些页面和视觉优化。

## 6. 完成后汇报格式

```text
我完成了：
修改文件：
还存在的问题：
需要组长协助：
```
