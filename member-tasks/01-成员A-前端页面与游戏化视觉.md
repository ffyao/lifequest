# 成员 A AI 任务书：前端页面与游戏化视觉

> 本任务书写给成员 A 的 AI 编程助手。请严格按本文档执行，不要越界修改后端核心逻辑。

## 1. 你的身份和目标

你是 LifeQuest 项目的前端视觉 AI。你的目标是优化页面观感和交互体验，让项目看起来像一个 RPG 人生副本系统，而不是普通 Todo List。

重点成果：

- 首页更有冲击力。
- 今日副本看板更适合答辩展示。
- 任务卡片区分主线、支线、每日、Boss。
- 徽章墙和排行榜更有游戏化氛围。
- 不破坏现有 API 和核心流程。

## 2. 通用执行流程

开始前请先按 `member-tasks/AI协作总则.md` 完成 SSH、clone、分支、基线测试检查。

你的工作分支是：

```text
feature/frontend-ui
```

本任务书只描述你的模块目标、文件范围和验收标准；Git 命令、提交、推送、PR 规则均以 `member-tasks/AI协作总则.md` 为准。

## 4. 任务 A1：首页视觉升级

位置：

```text
client/index.html
client/styles.css
```

目标：

- 强化 `LifeQuest 人生副本` 的第一屏视觉。
- 保留“开启副本”和“使用 Demo 账号”入口。
- 增加更明确的功能卖点展示。

建议方向：

- RPG HUD 面板。
- 发光按钮。
- 星球/副本/角色卡视觉。
- 深色渐变背景。
- 任务卷轴、经验条、徽章元素。

验收标准：

- 首页第一屏截图可直接放入 PPT。
- 不影响登录和页面锚点跳转。
- 移动端不出现明显横向溢出。

## 5. 任务 A2：今日副本看板优化

位置：

```text
client/index.html
client/styles.css
client/app.js
```

必须展示：

```text
角色昵称
职业标签
等级
XP
经验进度条
金币
连击天数
未完成任务数
已完成任务数
已解锁徽章数
NPC 鼓励语
```

要求：

- 数据必须来自现有接口，不要写死。
- 经验条根据 XP 动态变化。
- 看板布局适合答辩现场展示。

## 6. 任务 A3：任务卡片游戏化

位置：

```text
client/app.js
client/styles.css
```

需要区分四种任务：

```text
main      主线任务
side      支线任务
daily     每日任务
boss      Boss 任务
```

建议视觉：

```text
main  紫色/史诗
side  蓝色/探索
daily 绿色/日常
boss  金色或红色/挑战
```

每张卡片保留：

```text
任务标题
任务描述
任务类型
任务难度
XP 奖励
完成按钮
完成状态
```

验收标准：

- 点击完成按钮后，页面刷新并展示最新 XP。
- 已完成任务不可再次完成。
- Boss 任务视觉明显。

## 7. 任务 A4：徽章墙和排行榜优化

位置：

```text
client/app.js
client/styles.css
```

徽章墙要求：

- 已解锁徽章高亮。
- 未解锁徽章置灰。
- 展示徽章图标、名称、说明。

排行榜要求：

- 展示排名、昵称、职业、等级、XP。
- 前三名有特殊样式。
- 数据来自 `/api/ranking`。
