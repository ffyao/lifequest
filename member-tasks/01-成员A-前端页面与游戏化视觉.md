# 成员 A AI 任务书：前端页面与游戏化视觉

> 本任务书写给成员 A 的 AI 编程助手。请严格按本文档执行，不要越界修改后端核心逻辑。

## 1. 你的身份和目标

你是 LifeQuest 项目的前端视觉 AI。你的目标是优化页面观感和交互体验，让项目看起来像一个 RPG 人生副本系统，而不是普通 Todo List。

重点成果：

- 首页更有冲击力。
- 今日副本看板更适合答辩展示。
- 任务卡片区分主线、支线、每日、Boss。
- 徽章墙和排行榜更有游戏化氛围。
- 不破坏现有 API 和核心流程。

## 2. 开始前必须执行

成员账号已被组长添加为 Gitee 仓库管理员。成员 AI 具备终端和文件权限时，可以直接执行以下命令拉取仓库。请使用 SSH，不要改用 HTTPS。

如果当前还没有本地仓库，先执行：

```bash
git clone git@gitee.com:tidehope/lifequest.git
cd lifequest
```

```bash
git fetch origin
git checkout feature/frontend-ui
git pull --ff-only origin feature/frontend-ui
git status --short --branch
npm test
```

确认远程是 SSH：

```bash
git remote -v
```

必须看到：

```text
git@gitee.com:tidehope/lifequest.git
```

如果不是 SSH，或者 SSH 认证失败，不要继续开发，先反馈成员或组长。

## 4. 任务 A1：首页视觉升级

位置：

```text
client/index.html
client/styles.css
```

目标：

- 强化 `LifeQuest 人生副本` 的第一屏视觉。
- 保留“开启副本”和“使用 Demo 账号”入口。
- 增加更明确的功能卖点展示。

建议方向：

- RPG HUD 面板。
- 发光按钮。
- 星球/副本/角色卡视觉。
- 深色渐变背景。
- 任务卷轴、经验条、徽章元素。

验收标准：

- 首页第一屏截图可直接放入 PPT。
- 不影响登录和页面锚点跳转。
- 移动端不出现明显横向溢出。

## 5. 任务 A2：今日副本看板优化

位置：

```text
client/index.html
client/styles.css
client/app.js
```

必须展示：

```text
角色昵称
职业标签
等级
XP
经验进度条
金币
连击天数
未完成任务数
已完成任务数
已解锁徽章数
NPC 鼓励语
```

要求：

- 数据必须来自现有接口，不要写死。
- 经验条根据 XP 动态变化。
- 看板布局适合答辩现场展示。

## 6. 任务 A3：任务卡片游戏化

位置：

```text
client/app.js
client/styles.css
```

需要区分四种任务：

```text
main      主线任务
side      支线任务
daily     每日任务
boss      Boss 任务
```

建议视觉：

```text
main  紫色/史诗
side  蓝色/探索
daily 绿色/日常
boss  金色或红色/挑战
```

每张卡片保留：

```text
任务标题
任务描述
任务类型
任务难度
XP 奖励
完成按钮
完成状态
```

验收标准：

- 点击完成按钮后，页面刷新并展示最新 XP。
- 已完成任务不可再次完成。
- Boss 任务视觉明显。

## 7. 任务 A4：徽章墙和排行榜优化

位置：

```text
client/app.js
client/styles.css
```

徽章墙要求：

- 已解锁徽章高亮。
- 未解锁徽章置灰。
- 展示徽章图标、名称、说明。

排行榜要求：

- 展示排名、昵称、职业、等级、XP。
- 前三名有特殊样式。
- 数据来自 `/api/ranking`。

## 8. 推荐但非必须

如果时间允许，可以增加：

- Toast 样式优化。
- 页面响应式优化。
- 空状态提示。

不要增加复杂路由框架或大型依赖。
