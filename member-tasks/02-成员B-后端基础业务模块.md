# 成员 B 任务书：后端基础业务模块

## 1. 你的职责

你负责 LifeQuest 的后端基础业务接口和数据库 CRUD。目标是让前端可以稳定获取和保存用户、角色、目标、任务、排行榜等数据。

## 2. 你需要先了解

后端代码主要在：

```text
server/index.js
server/services/
```

数据库初始化在：

```text
server/services/database.js
```

API 路由在：

```text
server/services/api.js
```

## 3. 当前已有接口

```text
POST /api/auth/register
POST /api/auth/login
GET  /api/auth/me

GET  /api/character
POST /api/character
PUT  /api/character

GET  /api/goals
POST /api/goals
GET  /api/goals/:id

GET    /api/tasks
POST   /api/tasks/generate
PATCH  /api/tasks/:id/complete
DELETE /api/tasks/:id

GET /api/badges
GET /api/ranking
GET /api/ai/logs
```

## 4. 主要任务

### 4.1 补全目标接口

当前已有：

```text
GET /api/goals
POST /api/goals
GET /api/goals/:id
```

你可以补充：

```text
PUT    /api/goals/:id
DELETE /api/goals/:id
```

要求：

- 只能修改当前用户自己的目标。
- 删除目标时不要导致系统报错。
- 返回 JSON 格式统一。

### 4.2 补全任务编辑接口

当前已有：

```text
GET /api/tasks
PATCH /api/tasks/:id/complete
DELETE /api/tasks/:id
```

你可以补充：

```text
PUT /api/tasks/:id
```

支持修改：

```text
title
description
difficulty
dueDate
```

注意：

- 不要破坏完成任务后的 XP 计算逻辑。
- 不要让已完成任务重复获得 XP。

### 4.3 增加接口测试记录

建议用浏览器、Postman、Apifox 或 curl 测试。

至少测试：

```text
注册
登录
查询角色
创建目标
生成任务
完成任务
查询排行榜
```

需要提供接口测试截图给组长。

### 4.4 补充测试数据

可以在 `server/services/database.js` 中补充更多示例数据：

- 多个用户。
- 多个角色。
- 多个目标。
- 多个任务。
- 排行榜数据。

补充前先和组长确认，避免数据太乱。

## 5. 不建议你做

- 不要改 XP 和等级规则。
- 不要改徽章解锁规则。
- 不要改任务生成核心逻辑。
- 不要私自更改表名和字段名。

## 6. 你的交付物

- 后端接口代码。
- 接口测试截图。
- 简短说明：完成了哪些接口。

## 7. 验收标准

- `npm run dev` 能正常启动。
- `npm test` 能通过。
- 新增接口返回 JSON。
- 旧接口没有被破坏。
- 前端仍然能正常展示数据。
