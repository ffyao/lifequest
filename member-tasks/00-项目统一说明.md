# LifeQuest 项目统一说明

> 本文档写给成员和成员 AI。成员 AI 开始工作前，必须先阅读 `member-tasks/AI协作总则.md`，再阅读本文档和自己的任务书。

## 1. 项目基本信息

```text
项目名称：LifeQuest 人生副本任务系统
远程仓库：git@gitee.com:tidehope/lifequest.git
仓库网页：https://gitee.com/tidehope/lifequest
协作方式：Gitee + SSH + feature 分支 + Pull Request
```

组长已将其他成员账号添加为 Gitee 仓库管理员。成员 AI 具备终端和文件权限时，可以直接使用 SSH 拉取、开发、提交和推送代码。所有成员 AI 必须使用 SSH 方式连接 Gitee，不要切换为 HTTPS。

注意：SSH 配置与成员使用的系统环境相关。如果成员 AI 所在电脑尚未配置该成员的 Gitee SSH Key，需要先完成 SSH 配置或向组长反馈；不要擅自改用 HTTPS。

## 2. 项目一句话介绍

LifeQuest 把现实中的学习、健身、自律、创作等长期目标变成 RPG 游戏副本。用户创建人生角色后，可以输入长期目标，系统生成主线、支线、每日和 Boss 任务。用户完成任务后获得 XP、金币、等级提升和徽章奖励。

## 3. 当前技术栈

```text
前端：原生 HTML / CSS / JavaScript
后端：Node.js HTTP Server
数据库：Node.js 内置 SQLite
AI：本地模板兜底，可扩展真实大模型接口
测试：Node.js assert 核心流程测试
```

重要约束：

- 当前项目是零依赖原型，不需要 `npm install`。
- 不要擅自迁移到 Vue、React、Express、Prisma 等新技术栈。
- 如果确实需要新增依赖，必须在 PR 中说明原因、安装命令和影响。

## 4. 项目核心流程

任何成员 AI 的修改都不能破坏这个流程：

```text
登录 demo 账号
→ 读取人生角色
→ 输入长期目标
→ 生成副本任务
→ 展示任务列表
→ 完成任务
→ XP 增加
→ 等级变化
→ 徽章墙和排行榜正常显示
```

默认测试账号：

```text
用户名：demo
密码：demo123
```

## 6. 本地启动和测试

进入项目目录：

```bash
cd lifequest
```

运行核心测试：

```bash
npm test
```

启动项目：

```bash
npm run dev
```

浏览器访问：

```text
http://localhost:3000
```

## 7. Git 分支分配

```text
main                     稳定主分支，不直接开发
dev                      集成分支，不直接开发
feature/core             组长分支
feature/frontend-ui      成员 A 前端分支
feature/backend-crud     成员 B 后端分支
feature/ai-badges        成员 C AI/徽章/素材分支
```

成员 AI 必须只在对应 feature 分支开发，并向 `dev` 发 Pull Request。

```text

POST /api/auth/login
GET  /api/auth/me
```

角色：

```text
GET  /api/character
POST /api/character
PUT  /api/character
```

目标：

```text
GET  /api/goals
POST /api/goals
GET  /api/goals/:id
```

任务：

```text
GET    /api/tasks
POST   /api/tasks/generate
PATCH  /api/tasks/:id/complete
DELETE /api/tasks/:id
```

其他：

```text
GET /api/badges
GET /api/ranking
GET /api/ai/logs
```

## 9. 核心业务规则

任务类型：

```text
main      主线任务
side      支线任务
daily     每日任务
boss      Boss 任务
```

任务难度与奖励：

```text
easy      20 XP
normal    40 XP
hard      80 XP
boss      150 XP
```

等级规则：

```text
level = floor(xp / 100) + 1
```

## 10. AI 开发注意事项

成员 AI 应遵循：

- 先读文档，再改代码。
- 先运行 `npm test` 建立基线。
- 只改自己任务范围内的文件。
- 修改后再次运行 `npm test`。
- 不要提交运行时数据库文件。
- 不要隐藏失败或未完成项。
- 最终回复必须列出修改文件、测试结果、风险和下一步建议。
