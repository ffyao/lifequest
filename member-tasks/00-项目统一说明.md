# LifeQuest 项目统一说明

## 1. 项目名称

`LifeQuest 人生副本任务系统`

## 2. 项目一句话介绍

把现实中的学习、健身、自律等长期目标变成 RPG 游戏副本，用户通过完成任务获得经验值、升级角色并解锁徽章。

## 3. 项目核心流程

```text
注册/登录
→ 创建人生角色
→ 输入长期目标
→ 系统生成副本任务
→ 用户完成任务
→ 获得 XP 和金币
→ 等级提升
→ 解锁徽章
→ 查看排行榜
```

## 4. 当前技术栈

```text
前端：HTML/CSS/JavaScript
后端：Node.js
数据库：SQLite
AI：本地模板兜底，可扩展真实大模型
```

后续如果成员需要升级为 Vue，需要先和组长确认，避免影响当前可运行版本。

## 5. 项目目录

```text
lifequest/
├── client/                 前端页面
├── server/                 后端接口和数据库
├── docs/                   提交文档
├── member-tasks/           成员任务说明
├── screenshots/            截图材料
└── README.md
```

## 6. 启动方式

进入项目目录：

```bash
cd lifequest
npm run dev
```

浏览器打开：

```text
http://localhost:3000
```

默认账号：

```text
用户名：demo
密码：demo123
```

## 7. 开发协作规则

- 不要私自修改数据库字段。
- 不要私自修改 API 路径。
- 修改公共结构前先告诉组长。
- 每个人只改自己负责的模块。
- 完成后说明修改了哪些文件。
- 优先保证项目能跑，再优化细节。

## 8. API 简表

```text
POST /api/auth/register
POST /api/auth/login
GET  /api/auth/me

GET  /api/character
POST /api/character
PUT  /api/character

GET  /api/goals
POST /api/goals
GET  /api/goals/:id

GET    /api/tasks
POST   /api/tasks/generate
PATCH  /api/tasks/:id/complete
DELETE /api/tasks/:id

GET /api/badges
GET /api/ranking
GET /api/ai/logs
```

## 9. 数据规则

任务类型：

```text
main  主线任务
side  支线任务
daily 每日任务
boss  Boss 任务
```

任务难度：

```text
easy   20 XP
normal 40 XP
hard   80 XP
boss   150 XP
```

等级规则：

```text
level = floor(xp / 100) + 1
```
