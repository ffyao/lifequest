# 组长开发说明

## 1. 组长负责范围

本阶段组长已完成以下内容：

- 项目骨架搭建。
- 后端服务入口。
- SQLite 数据库初始化。
- 用户、角色、目标、任务、徽章、AI 日志数据模型。
- 核心 API 路由。
- 目标生成副本任务逻辑。
- 完成任务后 XP、金币、等级更新。
- 徽章自动解锁。
- 排行榜。
- 前端可演示页面。
- 核心流程测试。
- 课程提交文档初稿。
- 单人维护流程说明。

## 2. 核心链路

组长负责保证以下链路稳定：

```text
用户登录
→ 创建或读取角色
→ 创建目标
→ DeepSeek 生成任务
→ 任务写入数据库
→ 用户完成任务
→ 增加 XP 和金币
→ 重新计算等级
→ 判断徽章条件
→ 返回最新角色状态
```

## 3. 关键文件

```text
server/services/database.js      数据库表和默认数据
server/services/api.js           API 路由
server/services/aiService.js     DeepSeek 任务生成
server/services/taskService.js   任务生成和查询
server/services/gameService.js   XP、等级、任务完成
server/services/badgeService.js  徽章解锁
client/app.js                    前端接口调用和渲染
server/tests/core-flow.test.js   核心流程测试
```

## 4. 后续维护重点

后续由组长单人开发，需要重点检查：

- 是否破坏 `npm run dev`。
- 是否破坏 `npm test`。
- 是否改动公共 API 字段。
- 是否导致前端无法渲染。
- 是否影响 DeepSeek 任务生成和 API Key 配置流程。
- 是否同步更新操作手册、API 规范和 Prompt 文档。

## 5. 验收命令

```bash
npm test
```

通过后再启动项目：

```bash
npm run dev
```

访问：

```text
http://localhost:3000
```

注意不要访问 `https://localhost:3000`。当前开发服务只支持 HTTP，误用 HTTPS 会触发浏览器 SSL 报错。
