# LifeQuest 审查报告

## 1. 审查目的

本报告记录项目代码与功能的 AI 辅助审查过程，展示质量保障环节。审查重点包括核心业务流程、接口设计、数据一致性、范围控制和演示稳定性。

## 2. 审查范围

本次审查覆盖以下模块：

- `server/index.js`：服务入口和静态资源处理。
- `server/services/database.js`：数据库表结构和种子数据。
- `server/services/api.js`：API 路由分发。
- `server/services/taskService.js`：任务生成和任务查询。
- `server/services/gameService.js`：任务完成、XP、等级和金币规则。
- `server/services/badgeService.js`：徽章解锁规则。
- `server/services/aiService.js`：AI/模板生成逻辑。
- `client/app.js`：前端接口调用和页面渲染。
- `client/styles.css`：基础视觉样式。
- `server/tests/core-flow.test.js`：核心流程测试。

## 3. 审查方法

采用 AI 辅助代码审查与本地测试结合的方式。

审查 Prompt 示例：

```text
请审查 LifeQuest 人生副本任务系统的核心代码，重点检查：
1. 注册、创建角色、创建目标、生成任务、完成任务、经验升级、徽章解锁这条链路是否完整。
2. API 返回格式是否统一。
3. 数据库字段是否能支撑课程文档要求。
4. 是否存在范围失控或演示不稳定风险。
5. 给出严重程度、问题位置和修复建议。
```

本地验证命令：

```bash
npm test
```

## 4. 发现的问题与处理情况

| 编号 | 严重程度 | 位置 | 问题描述 | 处理情况 |
| --- | --- | --- | --- | --- |
| 1 | 中 | 用户登录 | 演示版本使用明文密码，不适合生产环境 | 已在操作手册和规格说明中说明，课程原型可接受 |
| 2 | 中 | 登录态 | 旧版本使用用户 ID 模拟身份，未登录也可能访问业务接口 | 已改为 Bearer token 登录态，业务接口未登录返回 `401` |
| 3 | 低 | AI 模块 | 当前未接入真实大模型 API | 已实现本地模板兜底，保留 `ai_logs` 便于后续扩展 |
| 4 | 低 | 前端 | 当前为原生前端原型，组件化程度有限 | 已规划成员 A 可继续升级为 Vue 页面 |
| 5 | 低 | 连击计算 | 当前连击天数按已完成日期数量简化统计 | 演示阶段可接受，后续可改为严格连续日期算法 |
| 6 | 中 | `client/app.js` | 登录后用户名通过 `innerHTML` 直接插入，存在前端注入风险 | 已改为使用 `escapeHtml` 转义用户名 |
| 7 | 低 | `gameService.completeTask` | 前端期望显示本次获得 XP，但后端未返回 `xpGained` | 已补充 `xpGained`，重复完成任务返回 `0` |
| 8 | 低 | `taskService.remove` | 删除不存在或不属于当前用户的任务时只返回 `deleted:false`，与其他接口错误语义不一致 | 已改为返回 `404 TASK_NOT_FOUND`，并补充测试 |
| 9 | 低 | `httpUtils.sendJson` | CORS 允许头需要支持登录态请求 | 已允许 `Authorization` 请求头 |
| 10 | 低 | `server/index.js`、操作文档 | 用户误用 `https://localhost:3000` 访问本地 HTTP 服务时，浏览器显示 SSL 报错 | 已补充启动提示、终端诊断提示和操作手册排错说明 |
| 11 | 中 | `userService`、`client/app.js` | 注册流程缺少激活码控制，无法限制账号创建 | 已加入普通/高级激活码机制和管理员管理页面 |

## 5. 已修复或规避的问题

- 核心任务完成逻辑避免重复获得 XP。
- 完成任务接口返回 `xpGained`，前端提示能显示本次获得经验。
- 登录状态提示中的用户名已转义，降低前端注入风险。
- 业务接口改为登录后携带 Bearer token，未登录无法使用核心系统功能。
- 注册账号必须填写激活码，普通码使用后自动从有效列表中移除。
- 管理员可生成普通或高级激活码，并查看高级码剩余次数。
- 删除不存在任务时返回统一的 `TASK_NOT_FOUND` 错误。
- CORS 允许 `Authorization` 请求头，支持 token 登录态。
- 本地服务启动提示明确区分 HTTP/HTTPS，并在误用 HTTPS 时给出终端诊断。
- 任务生成失败风险通过本地模板兜底规避。
- 数据库初始化时自动创建表和默认数据，降低演示环境风险。
- 提供 `npm test` 验证核心闭环。
- 首页、任务列表、徽章墙和排行榜均可基于真实接口渲染。

## 7. 审查结论

项目核心链路完整，能满足课程提交中的“项目完成质量”和“项目运行证据”要求。当前版本更偏向稳定演示原型，适合在此基础上由成员继续补充视觉页面、后端 CRUD、AI 文案和截图材料。

无阻塞级重大风险。需要在答辩时明确说明：密码存储和本地 AI 模板属于课程原型简化，正式产品需要进一步加强密码哈希、会话过期策略和模型接入。
