# LifeQuest 审查报告

## 1. 审查目的

本报告记录项目代码与功能的 AI 辅助审查过程，展示质量保障环节。审查重点包括核心业务流程、接口设计、数据一致性、范围控制和演示稳定性。

## 2. 审查范围

本次审查覆盖以下模块：

- `server/index.js`：服务入口和静态资源处理。
- `server/services/database.js`：数据库表结构和种子数据。
- `server/services/api.js`：API 路由分发。
- `server/services/taskService.js`：任务生成和任务查询。
- `server/services/gameService.js`：任务完成、XP、等级和金币规则。
- `server/services/badgeService.js`：徽章解锁规则。
- `server/services/aiService.js`：DeepSeek 任务生成逻辑。
- `client/app.js`：前端接口调用和页面渲染。
- `client/styles.css`：基础视觉样式。
- `server/tests/core-flow.test.js`：核心流程测试。

## 3. 审查方法

采用 AI 辅助代码审查与本地测试结合的方式。

审查 Prompt 示例：

```text
请审查 LifeQuest 人生副本任务系统的核心代码，重点检查：
1. 注册、创建角色、创建目标、生成任务、完成任务、经验升级、徽章解锁这条链路是否完整。
2. API 返回格式是否统一。
3. 数据库字段是否能支撑课程文档要求。
4. 是否存在范围失控或演示不稳定风险。
5. 给出严重程度、问题位置和修复建议。
```

本地验证命令：

```bash
npm test
```

## 4. 发现的问题与处理情况

| 编号 | 严重程度 | 位置 | 问题描述 | 处理情况 |
| --- | --- | --- | --- | --- |
| 1 | 中 | 用户登录 | 演示版本使用明文密码，不适合生产环境 | 已在操作手册和规格说明中说明，课程原型可接受 |
| 2 | 中 | 登录态 | 旧版本使用用户 ID 模拟身份，未登录也可能访问业务接口 | 已改为 7 天有效的 Bearer token 登录态，业务接口未登录返回 `401`，支持退出登录删除会话 |
| 3 | 低 | AI 模块 | 旧版本未接入真实大模型 API | 已接入 DeepSeek `deepseek-v4-flash`，使用用户级 API Key，并保留 `ai_logs` 记录生成过程 |
| 4 | 低 | 前端 | 当前为原生前端原型，组件化程度有限 | 后续由组长按需继续优化页面结构和交互 |
| 5 | 低 | 连击计算 | 旧版本按已完成日期数量简化统计 | 已改为从今天开始向前计算严格连续完成天数 |
| 6 | 中 | `client/app.js` | 登录后用户名通过 `innerHTML` 直接插入，存在前端注入风险 | 已改为使用 `escapeHtml` 转义用户名 |
| 7 | 低 | `gameService.completeTask` | 前端期望显示本次获得 XP，但后端未返回 `xpGained` | 已补充 `xpGained`；主线、支线、Boss 重复完成返回 `0`，每日任务同一天重复完成返回 `0` |
| 8 | 低 | `taskService.remove` | 删除不存在或不属于当前用户的任务时只返回 `deleted:false`，与其他接口错误语义不一致 | 已改为返回 `404 TASK_NOT_FOUND`，并补充测试 |
| 9 | 低 | `httpUtils.sendJson` | CORS 允许头需要支持登录态请求 | 已允许 `Authorization` 请求头 |
| 10 | 低 | `server/index.js`、操作文档 | 用户误用 `https://localhost:3000` 访问本地 HTTP 服务时，浏览器显示 SSL 报错 | 已补充启动提示、终端诊断提示和操作手册排错说明 |
| 11 | 中 | `userService`、`client/app.js` | 注册流程缺少激活码控制，无法限制账号创建 | 已加入普通/高级激活码机制和管理员管理页面 |
| 12 | 中 | `aiService` | 模型容易返回模板化标题或固定格式，影响任务可读性 | 已强化 Prompt，增加标题格式校验；可修复错误会携带失败原因重试一次 |
| 13 | 低 | `client/app.js`、`gameService` | 头像只能用短文本，展示不够直观 | 已改为点击头像选择图片文件，前端裁剪压缩后保存为 `data:image/...`，排行榜同步展示图片头像 |
| 14 | 中 | 排行榜 | 曾允许修改展示用户名，可能与账号身份不一致 | 已取消用户名修改机制，排行榜始终显示账号注册用户名 |

## 5. 已修复或规避的问题

- 核心任务完成逻辑避免重复获得 XP。
- 完成任务接口返回 `xpGained`，前端提示能显示本次获得经验。
- 登录状态提示中的用户名已转义，降低前端注入风险。
- 业务接口改为登录后携带 Bearer token，未登录无法使用核心系统功能，会话有效期为 7 天。
- 前端默认呈现独立登录页，登录成功后进入系统，顶部提供“退出登录”按钮。
- 注册账号必须填写激活码，普通码使用后自动从有效列表中移除。
- 管理员可生成普通或高级激活码，并查看高级码剩余次数。
- 删除不存在任务时返回统一的 `TASK_NOT_FOUND` 错误。
- CORS 允许 `Authorization` 请求头，支持 token 登录态。
- 本地服务启动提示明确区分 HTTP/HTTPS，并在误用 HTTPS 时给出终端诊断。
- 任务生成改为 DeepSeek `deepseek-v4-flash`，后端会校验模型返回 JSON、必选类型、可选支线、难度匹配和模板化标题，失败时不写入任务，也不回退到本地模板。
- 每日任务改为按日期完成记录，每天可完成一次，同一天重复点击不会重复获得 XP。
- 目标通关改为只依据主线和 Boss 任务，通关后该目标下所有完成按钮显示“已通关”。
- 头像上传改为图片文件选择，用户名保持账号注册用户名，排行榜同步头像、等级和 XP。
- 数据库初始化时自动创建表和默认数据，降低演示环境风险。
- 提供 `npm test` 验证核心闭环。
- 首页、目标列表、目标任务中心、徽章墙和排行榜均可基于真实接口渲染。

## 7. 审查结论

项目核心链路完整，能满足课程提交中的“项目完成质量”和“项目运行证据”要求。当前版本更偏向稳定演示原型，适合推送到 `main` 作为课程展示版本。后续可以继续补充部署配置、截图材料和更细粒度的端到端 UI 测试。

无阻塞级重大风险。需要在答辩时明确说明：密码存储属于课程原型简化，正式产品需要进一步加强密码哈希、会话过期策略和模型调用安全。
