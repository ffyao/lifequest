# LifeQuest 操作手册

## 1. 环境要求

- Node.js `>= 22.5.0`
- npm `>= 10`
- 现代浏览器，如 Chrome、Edge

说明：项目使用 Node.js 内置 SQLite，因此不需要额外安装 MySQL 或 SQLite 命令行工具。

## 2. 安装步骤

进入项目目录：

```bash
cd lifequest
```

当前版本没有第三方依赖，可以直接运行。如果后续维护时新增依赖，则先执行：

```bash
npm install
```

## 3. 启动项目

开发模式：

```bash
npm run dev
```

启动成功后，终端会显示：

```text
LifeQuest server running at http://localhost:3000
请使用 http://localhost:3000 访问，不要使用 https://localhost:3000
```

浏览器访问：

```text
http://localhost:3000
```

注意：项目默认只启动本地 HTTP 服务。如果浏览器地址栏自动变成 `https://localhost:3000`，页面可能显示 `SSL_ERROR_RX_RECORD_TOO_LONG`，需要手动改回 `http://localhost:3000`。

## 4. 默认账号

```text
用户名：demo
密码：demo123
```

管理员账号：

```text
用户名：admin
密码：admin123456
```

默认高级激活码：

```text
LIFEQUEST-ADV-300
```

说明：注册新账号必须填写激活码。高级激活码最多可激活 300 个账号；普通激活码只能激活 1 个账号，被使用后不会再显示在有效普通激活码列表中。

访问系统时默认先进入登录页。登录成功后会话有效期为 7 天，到期后需要重新登录。

## 5. 功能操作指南

### 5.1 页面导航

项目现在采用多功能页布局，顶部导航可切换：

- 首页：项目介绍和快速入口。
- 副本看板：角色状态、头像、注册用户名、XP、金币、连击天数和任务概览。
- 目标生成：输入目标并生成副本任务。
- 目标列表：按时间查看目标，点击目标进入对应任务中心。
- 徽章墙：查看已解锁和未解锁徽章。
- 排行榜：查看小组用户 XP 排名。
- 激活码管理：管理员生成和查看激活码。

每次只展示当前选中的功能页，避免把所有功能堆在同一个页面中。

### 5.2 登录

1. 打开系统，默认会显示独立登录页。
2. 输入用户名和密码。
3. 点击“登录”。
4. 成功后会自动进入“副本看板”。

### 5.3 注册

1. 打开系统，默认会显示独立登录页。
2. 输入用户名和密码。
3. 输入管理员提供的有效激活码。
4. 点击“注册并进入应用”。
5. 成功后会自动进入“副本看板”。

### 5.4 退出登录

1. 登录成功后，顶部导航会显示“退出登录”按钮。
2. 点击“退出登录”后会清理本地登录状态并回到登录页。
3. 后端会同步删除当前会话；如果 7 天会话过期，也会自动要求重新登录。

### 5.5 创建目标并生成任务

1. 点击顶部“目标生成”。
2. 输入目标标题，例如 `30 天掌握 Vue 项目开发`。
3. 选择目标分类，例如 `学习`。
4. 输入目标描述。
5. 确认管理员已在“系统配置”中保存 DeepSeek API Key。
6. 点击“生成副本任务”。
7. 成功后会自动进入“目标列表”，并默认选中新生成的目标。

### 5.6 查看目标列表并完成任务

1. 点击顶部“目标列表”。
2. 左侧目标列表按创建时间倒序展示，最新目标在最上方。
3. 点击某个目标，右侧会进入该目标的任务中心。
4. 使用筛选按钮查看该目标下的主线、支线、每日或 Boss 任务。
5. 找到未完成任务，点击“完成”。
6. 系统会增加 XP 和金币。
7. 如果达到条件，会自动解锁徽章。
8. 如果该目标包含每日任务，每日任务每天可以完成一次，完成后当天显示“今日已完成”，次日可再次挑战。
9. 短期目标可能只包含主线、Boss 和支线任务，不一定包含每日任务。
10. 目标通关进度只取决于主线任务和 Boss 任务，全部完成后目标视为通关。
11. 目标通关后，该目标下所有任务按钮都会显示“已通关”。
12. 刷新页面后，系统会默认恢复上次选中的目标任务中心。

### 5.7 查看角色状态

点击顶部“副本看板”，可以查看：

- 当前头像。
- 注册用户名。
- 当前等级。
- 当前 XP。
- 金币数量。
- 连击天数。

在副本看板中修改头像：

- 点击角色头像。
- 在本机选择图片文件。
- 前端会将图片裁剪压缩为正方形头像并保存到后端。

账号用户名使用注册时填写的用户名，系统不提供修改用户名或个性签名的入口。头像保存后会同步显示到任务看板和排行榜。

### 5.8 查看徽章墙

点击顶部“徽章墙”，可以查看：

- 已解锁徽章。
- 未解锁徽章。
- 徽章说明。

### 5.9 查看排行榜

点击顶部“排行榜”，可以查看小组用户 XP 排名。排行榜会展示每个用户当前头像、账号注册用户名、等级和 XP。

### 5.10 管理激活码

1. 使用管理员账号 `admin / admin123456` 登录。
2. 点击顶部“激活码管理”。
3. 点击“生成普通激活码”可生成一次性激活码。
4. 点击“生成高级激活码”可生成 300 次激活码。
5. 页面会显示所有有效高级激活码的剩余次数，以及所有仍有效的普通激活码。

## 7. 常见问题

### 7.1 端口被占用

如果 `3000` 端口被占用，可以改用其他端口：

```bash
PORT=3001 npm run dev
```

然后访问：

```text
http://localhost:3001
```

### 7.2 页面数据异常

可以删除数据库文件后重启：

```bash
rm server/data/lifequest.sqlite
npm run dev
```

重启后系统会自动创建表和默认数据。

### 7.3 Node 版本过低

如果提示 `node:sqlite` 不存在，需要升级 Node.js 到 22.5 及以上版本。

### 7.4 DeepSeek API Key 未配置或不可用

当前版本使用 DeepSeek `deepseek-v4-flash` 生成副本任务。DeepSeek API Key 由管理员统一配置，普通用户无需填写密钥。

处理方法：

1. 管理员前往 `platform.deepseek.com` 获取可用 API Key。
2. 管理员登录后进入“系统配置”，在 DeepSeek 配置区保存 API Key。
3. 如果提示密钥无效、余额不足、请求过频或网络失败，请管理员在 DeepSeek 平台检查密钥权限、账户余额和网络状态。
4. 测试环境不会自动调用真实 DeepSeek，`npm test` 使用 mock 请求验证生成逻辑。

### 7.5 访问时提示 SSL_ERROR_RX_RECORD_TOO_LONG

原因是浏览器正在用 HTTPS 访问本地 HTTP 服务。解决方法：

1. 在地址栏输入完整地址 `http://localhost:3000`，不要省略 `http://`。
2. 如果使用了其他端口，例如 `PORT=3001 npm run dev`，访问 `http://localhost:3001`。
3. 如果 Firefox 开启了 HTTPS-Only Mode，可以对 `localhost` 临时关闭 HTTPS 强制升级，或换用普通 HTTP 地址重新打开。
