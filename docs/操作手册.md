# LifeQuest 操作手册

## 1. 环境要求

- Node.js `>= 22.5.0`
- npm `>= 10`
- 现代浏览器，如 Chrome、Edge

说明：项目使用 Node.js 内置 SQLite，因此不需要额外安装 MySQL 或 SQLite 命令行工具。

## 2. 安装步骤

进入项目目录：

```bash
cd lifequest
```

当前版本没有第三方依赖，可以直接运行。如果后续成员添加依赖，则先执行：

```bash
npm install
```

## 3. 启动项目

开发模式：

```bash
npm run dev
```

启动成功后，终端会显示：

```text
LifeQuest server running at http://localhost:3000
请使用 http://localhost:3000 访问，不要使用 https://localhost:3000
```

浏览器访问：

```text
http://localhost:3000
```

注意：项目默认只启动本地 HTTP 服务。如果浏览器地址栏自动变成 `https://localhost:3000`，页面可能显示 `SSL_ERROR_RX_RECORD_TOO_LONG`，需要手动改回 `http://localhost:3000`。

## 4. 默认账号

```text
用户名：demo
密码：demo123
```

也可以在页面中注册新账号。

## 5. 功能操作指南

### 5.1 登录

1. 打开首页。
2. 在登录区域输入用户名和密码。
3. 点击“登录”。
4. 成功后页面会显示当前用户信息。

### 5.2 创建目标并生成任务

1. 找到“输入长期目标，生成副本任务”区域。
2. 输入目标标题，例如 `30 天掌握 Vue 项目开发`。
3. 选择目标分类，例如 `学习`。
4. 输入目标描述。
5. 点击“生成副本任务”。
6. 系统会生成主线、支线、每日和 Boss 任务。

### 5.3 完成任务

1. 滚动到任务列表。
2. 找到未完成任务。
3. 点击“完成”按钮。
4. 系统会增加 XP 和金币。
5. 如果达到条件，会自动解锁徽章。

### 5.4 查看角色状态

在“角色面板”中可以查看：

- 角色昵称。
- 职业标签。
- 当前等级。
- 当前 XP。
- 金币数量。
- 连击天数。

### 5.5 查看徽章墙

在“徽章墙”区域可以查看：

- 已解锁徽章。
- 未解锁徽章。
- 徽章说明。

### 5.6 查看排行榜

在“排行榜”区域可以查看小组用户 XP 排名。

## 7. 常见问题

### 7.1 端口被占用

如果 `3000` 端口被占用，可以改用其他端口：

```bash
PORT=3001 npm run dev
```

然后访问：

```text
http://localhost:3001
```

### 7.2 页面数据异常

可以删除数据库文件后重启：

```bash
rm server/data/lifequest.sqlite
npm run dev
```

重启后系统会自动创建表和默认数据。

### 7.3 Node 版本过低

如果提示 `node:sqlite` 不存在，需要升级 Node.js 到 22.5 及以上版本。

### 7.4 AI 接口没有配置

当前版本使用本地模板模拟 AI 任务生成，不需要配置 API Key。后续如果接入真实大模型，可以在 `server/services/aiService.js` 中扩展。

### 7.5 访问时提示 SSL_ERROR_RX_RECORD_TOO_LONG

原因是浏览器正在用 HTTPS 访问本地 HTTP 服务。解决方法：

1. 在地址栏输入完整地址 `http://localhost:3000`，不要省略 `http://`。
2. 如果使用了其他端口，例如 `PORT=3001 npm run dev`，访问 `http://localhost:3001`。
3. 如果 Firefox 开启了 HTTPS-Only Mode，可以对 `localhost` 临时关闭 HTTPS 强制升级，或换用普通 HTTP 地址重新打开。
