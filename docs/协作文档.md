# LifeQuest 协作文档

## 1. 团队分工

### 组长：整体架构、文档说明、核心功能开发

负责内容：

- 项目选题和范围控制。
- 技术栈和目录结构设计。
- 数据库表设计。
- API 接口规范。
- 核心链路开发：目标生成任务、完成任务、XP 增加、等级更新、徽章解锁。
- 五份提交文档主稿。
- 最终联调、截图、打包和答辩主线。

### 成员 A：前端页面与游戏化视觉

负责内容：

- 首页、登录页、副本看板、任务列表、角色卡片、徽章墙、排行榜。
- RPG 风格 UI，包括经验条、任务卡片、发光按钮、升级反馈。
- 页面截图和前端说明。

### 成员 B：后端基础业务模块

负责内容：

- 用户、角色、目标、任务、排行榜基础接口。
- 数据库 CRUD。
- 测试数据。
- 接口测试截图。

### 成员 C：AI 扩展、成就系统与测试素材

负责内容：

- AI Prompt 设计。
- 本地模板数据扩展。
- 徽章和 NPC 文案设计。
- AI 交互截图、测试截图、PPT 素材整理。

## 2. 协作工具链

- AI 编程工具：Codex / ChatGPT 类大模型。
- 代码管理：本地项目目录，可后续使用 Git。
- 前端运行：浏览器。
- 后端运行：Node.js。
- 数据存储：SQLite。
- 文档：Markdown。
- 截图：浏览器截图和 AI 对话截图。

## 3. AI 协作流程

### 3.1 需求分析

使用 AI 对课程要求和评分标准进行分析，确定项目需要覆盖：

- 规格说明。
- 架构设计。
- 审查报告。
- 操作手册。
- 协作文档。
- AI 交互截图。
- 项目运行截图。

### 3.2 选题讨论

先讨论多个创意 Web 应用方向，包括：

- 情绪星球生成器。
- AI 梦境档案馆。
- 城市奇遇生成器。
- 人生副本任务系统。

最终选择“人生副本任务系统”，原因是：

- 比普通 Todo List 更有创意。
- 可以自然加入 AI 任务拆解。
- 有前端、后端、数据库和核心业务逻辑。
- 适合四人小组分工。

### 3.3 架构设计

使用 AI 辅助设计：

- 用户、角色、目标、任务、徽章、AI 日志数据模型。
- API 路径。
- 核心业务链路。
- 小组分工。
- 项目开发流程。

人工确认后，组长负责落实核心代码。

### 3.4 代码生成

使用 AI 辅助生成：

- Node.js 后端服务。
- SQLite 表结构。
- AI/模板任务生成服务。
- XP 和等级规则。
- 徽章解锁规则。
- 前端演示页面。
- 核心流程测试。

生成后由组长审查并运行测试。

### 3.5 代码审查

使用 AI 对核心链路进行审查，重点检查：

- 是否能完整跑通。
- 是否有重复加 XP 问题。
- 是否有接口不一致问题。
- 是否存在演示风险。
- 是否符合课程提交要求。

审查结果整理到 `审查报告.md`。

## 4. AI Prompt 示例

### 需求分析 Prompt

```text
请根据课程提交要求和评分标准，帮我分析一个四人小组 AI Coding 项目应该优先完成哪些交付物和功能，要求范围可控、适合答辩展示。
```

### 选题 Prompt

```text
我想做一个具有趣味和创意的 Web 应用，可以选择性使用 AI 大模型。请给出适合课程项目的选题建议，并分析创意、技术复杂度和演示效果。
```

### 架构 Prompt

```text
我们决定做人生副本任务系统。请设计项目功能模块、技术架构、数据库表、API 接口和四人小组分工。
```

### 代码审查 Prompt

```text
请审查 LifeQuest 项目的核心代码，检查注册、角色、目标、任务生成、任务完成、经验升级、徽章解锁链路是否完整，并指出风险和修复建议。
```

## 5. 协作效率对比

如果完全手写，需要先自行整理需求、设计架构、拆分数据库和接口，再编写文档，时间成本较高。使用 AI 辅助后：

- 选题和范围控制更快。
- 数据库和 API 初稿更完整。
- 文档结构能直接对齐评分标准。
- 代码原型可以快速跑通核心闭环。
- 审查报告和操作手册更容易整理。

但 AI 输出仍需要人工确认，尤其是：

- 功能范围是否过大。
- 接口字段是否统一。
- 代码是否能真实运行。
- 文档是否符合本项目实际情况。

## 6. 团队协作规则

- 组长负责公共结构和接口规范。
- 成员不得私自修改数据库字段和核心 API 路径。
- 每个成员提交前说明修改内容。
- 每天至少同步一次进度。
- 优先保证核心链路能跑通，再追求视觉和扩展功能。

## 7. 经验总结

本项目体现了 AI Coding 的典型流程：先由 AI 辅助完成需求分析和方案设计，再由组长确认范围并开发核心链路，之后成员并行补齐页面、接口、AI 文案和测试材料。AI 提高了项目启动速度，但最终质量仍依赖人工判断、测试和整合。
