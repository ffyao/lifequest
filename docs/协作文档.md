# LifeQuest AI 使用记录

## 1. 当前维护方式

本项目后续由组长单人维护。早期用于多人分工的成员任务提示词和成员任务目录已删除，后续开发以 `README.md`、`CONTRIBUTING.md` 和 `docs/开发流程.md` 为准。

## 2. AI 辅助开发方式

项目仍保留 AI Coding 过程记录，用于说明课程项目如何使用 AI 提升开发效率：

- 需求分析：使用 AI 梳理课程要求、功能范围和演示重点。
- 方案设计：使用 AI 设计 LifeQuest 的功能模块、数据模型、API 和页面结构。
- 代码生成：使用 AI 辅助生成 Node.js 服务、SQLite 表结构、前端页面和测试。
- 代码审查：使用 AI 审查认证、任务生成、任务完成、XP、徽章和文档一致性。
- 文档整理：使用 AI 维护 README、操作手册、API 规范、架构说明和 Prompt 设计文档。

## 4. AI 功能说明

LifeQuest 的产品侧 AI 功能是“人生副本任务生成”：

```text
用户输入长期目标
→ 首次生成时配置个人 DeepSeek API Key
→ 后端调用 DeepSeek deepseek-v4-flash
→ DeepSeek 返回结构化任务 JSON
→ 后端校验任务数量、类型、难度和文本长度
→ 写入任务表和 AI 日志
→ 前端展示主线、支线、每日和 Boss 任务
```

关键约束：

- 每次生成必须得到 5 个任务。
- 类型组合为 1 个主线、1 个支线、2 个每日、1 个 Boss。
- Boss 任务难度必须为 `boss`。
- XP 由后端根据难度计算，不由模型直接决定。
- 生成失败不写入任务。

## 7. 经验总结

AI 提高了项目启动、方案设计、代码生成和文档整理速度，但最终质量仍依赖人工判断、测试和整合。后续维护应继续坚持“小步开发、及时测试、同步文档”的原则。
