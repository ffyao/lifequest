# LifeQuest 架构设计

## 1. 技术栈选型

| 层级 | 技术 | 选型理由 |
| --- | --- | --- |
| 前端 | HTML/CSS/JavaScript | 零依赖，便于快速启动、演示和维护 |
| 后端 | Node.js HTTP Server | 轻量、启动简单、适合课程原型 |
| 数据库 | Node.js 内置 SQLite | 无需额外安装数据库，能体现数据持久化 |
| AI | DeepSeek `deepseek-v4-flash` | 使用官方对话补全接口生成结构化副本任务 |
| 文档 | Markdown | 易编辑、易提交、易转 PDF |

当前实现采用零依赖可运行原型，降低课程演示和本地运行风险；后续如需要工程化升级，可以在 `client/` 和 `server/` 现有接口基础上迁移到 Vue、Express 或其他框架。当前发布版以稳定演示和文档完整为优先。

## 5. 核心 API 设计

### 用户

```text
POST /api/auth/register
POST /api/auth/login
GET  /api/auth/me
PUT  /api/profile
```

### 角色

```text
GET  /api/character
POST /api/character
PUT  /api/character
```

### 目标

```text
GET    /api/goals
POST   /api/goals
GET    /api/goals/:id
PUT    /api/goals/:id
DELETE /api/goals/:id
```

### 任务

```text
GET    /api/tasks
POST   /api/tasks/generate
PUT    /api/tasks/:id
PATCH  /api/tasks/:id/complete
DELETE /api/tasks/:id
```

### 徽章与排行榜

```text
GET /api/badges
GET /api/ranking
```

排行榜返回 `username`、`nickname`、`career`、`avatar`、`level`、`xp` 等展示字段。`username` 始终来自账号注册信息；副本看板点击头像后通过 `PUT /api/profile` 只更新 `characters.avatar`。

### AI 日志

```text
GET /api/ai/logs
```

## 6. 核心业务流程

### 6.1 任务生成流程

```text
前端提交目标和可选 DeepSeek API Key
→ 后端创建目标
→ AI 服务读取该用户已保存的 API Key 或使用本次提交的 API Key
→ 调用 DeepSeek deepseek-v4-flash 的 /chat/completions
→ 校验 JSON、任务数量、类型组合、难度、文本长度和模板化标题
→ 如果可修复的格式问题出现，携带失败原因重试一次
→ 校验通过后写入 ai_logs 表
→ 事务写入 tasks 表
→ 返回目标、任务列表和 NPC 文案
→ 前端保存当前用户选中的目标 ID
→ 目标列表按时间倒序展示，任务中心只展示当前目标任务
```

### 6.2 完成任务流程

```text
前端点击完成
→ 后端校验任务归属
→ 每日任务按 task_daily_completions 记录当天完成；非每日任务更新 status=done
→ 若任务已完成或今日已完成则返回 xpGained=0
→ 首次完成时根据 xpReward 增加角色 XP
→ 重新计算等级、金币和严格连续天数
→ 检查徽章条件
→ 若主线和 Boss 均完成则标记目标 completedAt
→ 返回新角色状态、新解锁徽章、目标状态和 xpGained
```


### 6.3 头像更新流程

```text
前端点击角色头像
→ 用户选择本地图片文件
→ 前端裁剪压缩为 160×160 WebP data URL
→ PUT /api/profile 只更新 characters.avatar
→ 用户名继续使用 users.username
→ 副本看板和排行榜重新渲染头像
```

## 7. 游戏化规则

### 任务难度与 XP

```text
easy   = 20 XP
normal = 40 XP
hard   = 80 XP
boss   = 150 XP
```

### 等级规则

```text
level = floor(xp / 100) + 1
```

### 徽章规则

- 首次启程：创建角色。
- 目标制定者：创建目标。
- 副本挑战者：完成第一个任务。
- 连击达人：连续完成 3 天任务。
- 主线推进者：完成 3 个主线任务。
- Boss 击破者：完成 1 个 Boss 任务。
- 经验收集者：累计 300 XP。
- 成长冒险家：等级达到 5 级。

## 8. 部署架构

### 本地开发

```text
浏览器 → localhost:3000 → Node.js → SQLite
```

启动命令：

```bash
npm run dev
```

### 生产演示

可部署到支持 Node.js 的平台。SQLite 文件随服务端保存；如果部署平台不支持持久文件系统，建议改用 MySQL 或 PostgreSQL。

## 9. AI 辅助说明

本项目 AI 功能分为两层：

- 产品功能层：目标到任务的生成逻辑，当前调用 DeepSeek `deepseek-v4-flash` 并进行后端结构校验。
- 开发过程层：需求分析、架构设计、代码生成、代码审查和文档编写使用 AI 辅助完成。

当前实现使用 DeepSeek `deepseek-v4-flash` 生成任务，并保留 `ai_logs` 记录输入、校验后的输出和模型名称。
