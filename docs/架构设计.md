# LifeQuest 架构设计

## 1. 技术栈选型

| 层级 | 技术 | 选型理由 |
| --- | --- | --- |
| 前端 | HTML/CSS/JavaScript | 零依赖，便于快速演示和成员后续迁移到 Vue |
| 后端 | Node.js HTTP Server | 轻量、启动简单、适合课程原型 |
| 数据库 | Node.js 内置 SQLite | 无需额外安装数据库，能体现数据持久化 |
| AI | 本地模板服务 | 保证演示稳定，可扩展真实大模型 API |
| 文档 | Markdown | 易编辑、易提交、易转 PDF |

原计划可使用 Vue 3 + Express + Prisma。当前组长核心部分先实现零依赖可运行原型，降低环境风险；成员后续可以在 `client/` 基础上继续升级前端页面。

## 6. 核心业务流程

### 6.1 任务生成流程

```text
前端提交目标
→ 后端创建目标
→ AI 模板服务匹配目标分类
→ 生成主线/支线/每日/Boss 任务
→ 写入 tasks 表
→ 写入 ai_logs 表
→ 返回任务列表和 NPC 文案
```

### 6.2 完成任务流程

```text
前端点击完成
→ 后端校验任务归属
→ 更新任务状态为 done
→ 根据 xpReward 增加角色 XP
→ 重新计算等级
→ 增加金币
→ 检查徽章条件
→ 返回新角色状态和新解锁徽章
```

## 7. 游戏化规则

### 任务难度与 XP

```text
easy   = 20 XP
normal = 40 XP
hard   = 80 XP
boss   = 150 XP
```

### 等级规则

```text
level = floor(xp / 100) + 1
```

### 徽章规则

- 首次启程：创建角色。
- 目标制定者：创建目标。
- 副本挑战者：完成第一个任务。
- 连击达人：连续完成 3 天任务。
- 主线推进者：完成 3 个主线任务。
- Boss 击破者：完成 1 个 Boss 任务。
- 经验收集者：累计 300 XP。
- 成长冒险家：等级达到 5 级。

## 8. 部署架构

### 本地开发

```text
浏览器 → localhost:3000 → Node.js → SQLite
```

启动命令：

```bash
npm run dev
```

### 生产演示

可部署到支持 Node.js 的平台。SQLite 文件随服务端保存；如果部署平台不支持持久文件系统，建议改用 MySQL 或 PostgreSQL。

## 9. AI 辅助说明

本项目 AI 功能分为两层：

- 产品功能层：目标到任务的生成逻辑，当前用本地模板模拟大模型结构化输出。
- 开发过程层：需求分析、架构设计、代码生成、代码审查和文档编写使用 AI 辅助完成。

当前实现保留了 `ai_logs`，便于后续接入真实大模型 API 后记录输入输出。
