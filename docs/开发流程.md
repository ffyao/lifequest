# LifeQuest 开发流程

## 1. 开发总原则

项目后续由组长单人维护，优先保证核心演示链路稳定。开发优先级如下：

```text
核心链路能跑通
→ 认证和激活码可用
→ 页面能展示
→ 数据能保存
→ DeepSeek 任务能生成
→ 文档和截图完整
```

## 2. 当前维护流程

### 2.1 需求确认

- 明确本次改动目标。
- 判断是否影响 API、数据库、认证、任务生成、XP 或徽章规则。
- 如果影响公共结构，同步更新测试和文档。

### 2.2 本地开发

- 在 `dev` 分支开发。
- 较大功能可临时创建 `feature/*` 分支，完成后合并回 `dev`。
- 避免提交真实 DeepSeek API Key、Token 或个人隐私数据。

### 2.3 验证检查

常规检查：

```bash
node --check server/services/aiService.js
node --check server/services/taskService.js
node --check server/services/api.js
node --check server/services/gameService.js
node --check client/app.js
git diff --check
npm test
```

如改动前端页面，还需手动访问：

```text
http://localhost:3000
```

注意本地服务是 HTTP，不要访问 `https://localhost:3000`。

### 2.4 文档同步

以下文件根据改动同步更新：

```text
README.md
docs/操作手册.md
docs/API接口规范.md
docs/架构设计.md
docs/AI-Prompt设计.md
docs/审查报告.md
```

## 3. 核心演示流程

```text
打开登录页
→ 登录或注册账号
→ 创建或查看人生角色
→ 可选：点击头像上传图片
→ 输入长期目标
→ 首次生成任务时配置 DeepSeek API Key
→ 生成副本任务
→ 在目标列表选择目标并进入任务中心
→ 完成任务，验证每日任务与通关状态
→ 展示 XP、等级、金币、徽章和排行榜变化
→ 总结 AI 辅助开发和 AI 功能
```

## 4. 改动规则

- API 路径变更必须同步 `docs/API接口规范.md`。
- 数据库字段变更必须同步 `docs/架构设计.md` 和测试。
- DeepSeek Prompt 或模型参数变更必须同步 `docs/AI-Prompt设计.md`。
- 任务完成逻辑必须保持幂等，主线、支线、Boss 不能重复获得 XP；每日任务同一天不能重复获得 XP。
- 前端 UI 不展示默认账号、管理员密码、激活码或 API Key 明文。
- 头像只能通过图片文件生成的 `data:image/...` 数据保存；用户名使用注册用户名，不提供修改入口。

## 5. 提交建议

提交信息保持清晰，例如：

```text
feat: add deepseek task generation
fix: prevent duplicate task rewards
docs: update single-maintainer workflow
```

提交前先查看：

```bash
git status --short
git diff --stat
```


## 6. 准备推送 main 的发布流程

发布到 `main` 前按以下顺序执行：

```bash
git checkout dev
git status --short
node --check client/app.js
node --check server/services/api.js
node --check server/services/aiService.js
node --check server/services/gameService.js
node --check server/services/taskService.js
node --check server/services/userService.js
node --check server/tests/core-flow.test.js
git diff --check
npm test
```

确认通过后再合并：

```bash
git checkout main
git merge dev
git push origin main
```

如果还需要保留远程 `dev` 最新状态，也可以在合并后执行：

```bash
git checkout dev
git push origin dev
```

推送前不要提交真实 DeepSeek API Key、浏览器截图中的敏感信息或本地数据库文件。
