# LifeQuest 开发流程

## 1. 开发总原则

项目采用“组长掌控主链路，成员承包子模块”的方式开发。优先级如下：

```text
核心链路能跑通
→ 页面能展示
→ 数据能保存
→ AI/模板能生成
→ 视觉效果好看
→ 文档截图完整
```

## 2. 阶段安排

### 阶段一：项目启动

负责人：组长

任务：

- 确定项目名称和范围。
- 搭建项目目录。
- 确定技术栈。
- 建立文档目录。
- 明确成员任务。

产出：

- `README.md`
- 项目骨架
- 成员任务说明

### 阶段二：架构冻结

负责人：组长

任务：

- 确定数据库表。
- 确定 API 路径。
- 确定前端页面路由。
- 确定任务、XP、等级、徽章规则。

产出：

- `docs/架构设计.md`
- API 接口规范
- 数据库初始化代码

### 阶段三：并行开发

负责人：全体成员

任务：

- 组长开发核心链路。
- 成员 A 开发前端页面。
- 成员 B 开发后端 CRUD。
- 成员 C 准备 AI 扩展和素材。

产出：

- 可运行的前后端代码。
- AI Prompt 和模板。
- 页面和接口截图。

### 阶段四：第一次联调

负责人：组长

必须跑通：

```text
登录
→ 创建角色
→ 创建目标
→ 生成任务
→ 完成任务
→ XP 增加
→ 等级更新
→ 徽章解锁
```

产出：

- 核心流程测试通过。
- 关键页面可截图。

### 阶段五：视觉和功能增强

负责人：成员 A、成员 C，组长把关

任务：

- 优化 RPG 风格页面。
- 增加 NPC 文案。
- 完善徽章墙。
- 完善排行榜。
- 整理 AI 交互记录。

产出：

- 演示效果更强的页面。
- PPT 可用素材。

### 阶段六：测试与审查

负责人：组长、成员 C

任务：

- 运行核心测试。
- 使用 AI 进行代码审查。
- 记录发现问题和修复情况。
- 完成项目截图。

产出：

- `docs/审查报告.md`
- 测试截图
- 运行截图

### 阶段七：提交打包

负责人：组长

任务：

- 检查五份文档。
- 检查代码是否可运行。
- 整理截图。
- PPT 转 PDF。
- 打包提交。

产出：

- `第X组-姓名.zip`
- `第X组-姓名.pdf`

## 3. 每日同步模板

每个成员同步时按以下格式：

```text
今天完成：
遇到问题：
需要组长协助：
下一步：
```

## 4. 改动规则

- 公共 API 路径由组长统一修改。
- 数据库字段由组长统一修改。
- 成员 A 主要修改 `client/`。
- 成员 B 主要修改 `server/services/` 中后端业务文件。
- 成员 C 主要修改 AI 文案、模板、徽章配置和截图材料。
- 提交前说明修改文件和影响范围。

## 5. 最终演示流程

```text
展示首页
→ 登录 demo 账号
→ 展示角色面板
→ 输入长期目标
→ 生成副本任务
→ 完成一个任务
→ 展示 XP 和等级变化
→ 展示徽章墙
→ 展示排行榜
→ 总结 AI 辅助开发和 AI 功能
```
